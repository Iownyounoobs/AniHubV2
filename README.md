## `> WHAT IS ANIHUB?`

AniHub is a full-stack anime streaming platform built from scratch as a CS capstone. It aggregates content from **AniWatchTV** through a custom scraping API and delivers it through a cyberpunk-themed React frontend тАФ complete with user auth, a personal watchlist, and Stripe-powered premium subscriptions.

Three independent servers. One cohesive experience.

> **Educational Use Only** тАФ Built as a capstone for Metropolitan State University. Not intended for public distribution or commercial use.

## `> FEATURES`

| Feature | Description |
|---|---|
| **Search & Browse** | Search by title, browse by genre, or filter by origin (Japan / China) |
| **Inline Episode Streaming** | MegaCloud sources iframed directly тАФ plays in-page with a custom overlay player |
| **MegaCloud Decryption** | Custom Crypto-JS pipeline decrypts MegaCloud stream sources on the backend |
| **My List** | Save/remove shows to a personal watchlist stored in Firestore |
| **Premium Access** | Stripe subscription gates Movies and full content тАФ monthly or yearly plan |
| **Auth & Role System** | Firebase Auth + Firestore roles (`isPremium`) with protected routes |
| **Chinese Donghua** | Dedicated `/origin/china` page using studio-keyword detection to filter Chinese anime |
| **Category Browsing** | Top-Airing, Most Popular, Genre pages, Subtype filtering |
| **Latest Episodes** | Live feed of recently updated episodes |
| **Image & M3U8 Proxy** | Backend proxies images and M3U8 playlists to bypass CORS |
| **Rate Limiting & Caching** | `express-rate-limit` + `node-cache` + `p-limit` for stable scraping |

## `> API REFERENCE`

All routes served from `http://localhost:3001`

```
GET  /                                            тЖТ Health check + AniWatchTV status
GET  /aniwatchtv                                  тЖТ Homepage: spotlight, top10, trending, genres
GET  /aniwatchtv/search?keyword=X&page=Y          тЖТ Search results with pagination
GET  /aniwatchtv/anime/:id                        тЖТ Full anime metadata
GET  /aniwatchtv/episodes/:id                     тЖТ Episode list
GET  /aniwatchtv/servers                          тЖТ Available streaming servers
GET  /aniwatchtv/episode-srcs                     тЖТ Decrypted MegaCloud stream URL
       ?id=X&ep=Y&server=MegaCloud&category=sub
GET  /aniwatchtv/latest                           тЖТ Recently updated episodes
GET  /aniwatchtv/:category                        тЖТ Category pages (top-airing, most-popularтАж)
GET  /aniwatchtv/genre/:genre                     тЖТ Genre-filtered anime
GET  /aniwatchtv/subtype/:subtype                 тЖТ Subtype-filtered anime
GET  /aniwatchtv/donghua?page=Y                   тЖТ Chinese anime (studio-keyword filtered)
GET  /aniwatchtv/img?url=X                        тЖТ Image proxy (CORS bypass)
GET  /aniwatchtv/proxy?url=X                      тЖТ M3U8 stream proxy
```

## `> INTERNAL ARCHITECTURE тАФ API`

The scraping API follows a strict layered pattern:

```
Route тЖТ Controller тЖТ Scraper тЖТ Extractor (Cheerio) тЖТ Typed Response
```

```
src/
тФЬтФАтФА routes/          Express router definitions
тФЬтФАтФА controllers/     Request handlers тАФ orchestrate scraper calls
тФЬтФАтФА scrapers/        Fetch + parse aniwatchtv.to pages
тФЬтФАтФА extractors/      Cheerio selectors that pull specific data shapes
тФЬтФАтФА types/           TypeScript interfaces for every response shape
тФЬтФАтФА utils/           MegaCloud pipeline, proxies, caching, status checks
тФФтФАтФА config/          HTTP headers (User-Agent, cookies, etc.)
```

Building AniHub solo taught me things no classroom could fully replicate:

- **Full-stack separation** тАФ keeping UI, API, and payments as independent servers forces clean interfaces

## `> RUNNING LOCALLY`

You need three terminals.

```bash
# Terminal 1 тАФ React UI
cd anihub-ui
npm install
npm start
# тЖТ http://localhost:3000

# Terminal 2 тАФ Anime API
cd aniwatchtv-api
npm install
npm run dev
# тЖТ http://localhost:3001

# Terminal 3 тАФ Stripe Backend
cd stripe-backend
npm install
node index.js
# тЖТ http://localhost:3002
```

**Required `.env` files:**

`anihub-ui/.env`
```
REACT_APP_FIREBASE_API_KEY=
```
